Building the App: Automation

Naturally, we don't want to have to manually run the script every day. The next step in the process is to use some sort of task scheduler to run our script for us. There are lots of options, but it seems like each operating system has its own standard one:

  • For Windows, there's Windows Task Scheduler
  • For MacOS, there's launchd
  • For Linux, there's cron

There are some guides about how to use it—you're more or less on your own here but there seem to be a lot of resources. The main things you should try to find out are:

  • How to start the job ASAP if your computer happens to be off (this might help)
  • How to detect if the job has failed (e.g. if you don't have internet connection or something), and have it try again. You'll probably want to add some error-checking and handling after each request in update.py.
