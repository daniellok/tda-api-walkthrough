Requirements

Before we get started, you'll need to have a few things installed.

Git Bash

We'll be using Git as a source-control tool. Source control is how we keep track of the changes we make to our code. Git allows us to do stuff like back up old versions of our code, see line-by-line differences between versions, and collaborate. This is the standard way that programmers build projects.

Git Bash is just an installation of Git for Windows. It provides a Unix-like terminal (i.e. the ones used on MacOS and Linux) so you don't have to worry about the commands being different. In this walkthrough, when I mention the "terminal" or the "command line", I mean that you should enter the commands in the Git Bash terminal. Unix-like terminals typically have a dollar sign ($) at the place where you're supposed to type commands, so if you see me type something like:


$ echo "hello"
hello
$ echo "goodbye"
goodbye

The stuff after the $ is what you should be typing. The stuff under it is the output you should receive.

Python 3

This is our main working language. You probably already have it installed, but in case you don't, go get it!

Requests

Requests is a third-party Python package (also known as a library) used to handle HTTP requests. Since it's not included with Python by default, we'll have to install it separately. Thankfully, this is super easy in Python.

Python comes with a package manager called pip. A package manager is, as the name implies, a program to install, uninstall, and update packages.
